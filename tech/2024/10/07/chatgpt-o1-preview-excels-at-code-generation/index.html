// Function to swap two integers
void swap(int& a, int& b) {
    int temp = a;
    a = b;
    b = temp;
}

// Partition function used by QuickSort
int partition(int arr[], int low, int high) {
    int pivot = arr[high]; // Choosing the last element as the pivot
    int i = (low - 1);     // Index of the smaller element

    for (int j = low; j <= high - 1; j++) {
        // If the current element is smaller than the pivot
        if (arr[j] < pivot) {
            i++;          // Increment index of smaller element
            swap(arr[i], arr[j]); // Swap
        }
    }
    swap(arr[i + 1], arr[high]); // Place pivot at the correct position
    return (i + 1);
}

// QuickSort function
void quickSort(int arr[], int low, int high) {
    if (low < high) {
        // Partitioning index
        int pi = partition(arr, low, high);

        // Recursively sort elements before and after partition
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}

Explanation of the QuickSort Algorithm:

  • Divide and Conquer: QuickSort is a divide-and-conquer algorithm. It divides the input array into two smaller sub-arrays based on a pivot element.
  • Pivot Selection: In this implementation, the last element is chosen as the pivot.
  • Partitioning: Rearranges the elements so that all elements less than the pivot come before it, while all greater elements come after.
  • Recursive Sorting: The sub-arrays are then sorted recursively.

Time Complexity:

  • Average Case: O(n log n)
  • Worst Case: O(n^2) (when the smallest or largest element is always chosen as the pivot)

Space Complexity:

  • O(log n) due to recursive function calls on the stack.

Advantages of QuickSort:

  • Generally faster in practice compared to other O(n log n) algorithms like Merge Sort.
  • In-place sorting (requires only a small amount of extra memory space).

Disadvantages of QuickSort:

  • Not stable (the relative order of equal elements may not be preserved).
  • Worst-case performance can be poor without careful pivot selection.

            bool doParallel = (thread_count < MAX_THREADS) && ((lt - low > PARALLEL_THRESHOLD) || (high - gt > PARALLEL_THRESHOLD));

            if (doParallel) {
                thread_count += 2; // We're going to spawn two new threads

                // Launch two threads to sort the partitions in parallel
                std::thread t1([&arr, low, lt, &rng]() {
                    quickSort(arr, low, lt - 1, rng);
                    thread_count--;
                });
                std::thread t2([&arr, gt, high, &rng]() {
                    quickSort(arr, gt + 1, high, rng);
                    thread_count--;
                });

                t1.join();
                t2.join();
                break; // Both partitions are sorted, exit the loop
            } else {
                // Sort smaller partition first to optimize space
                if (lt - low < high - gt) {
                    quickSort(arr, low, lt - 1, rng);
                    low = gt + 1; // Tail call elimination
                } else {
                    quickSort(arr, gt + 1, high, rng);
                    high = lt - 1; // Tail call elimination
                }
            }
        }
    }
}

        if (arr[i - 1] > arr[i])
            return false;
    return true;
}

    auto start = std::chrono::high_resolution_clock::now();

    if (arr.size() >= 1000)
        std::cout << " (" << duration / 1000.0 << " milliseconds)";
    std::cout << std::endl;

    std::cout << "Array is sorted: " << (isSorted(arr) ? "Yes" : "No") << "\n" << std::endl;
}

Explanation of Parallelization:

  1. Added std::atomic thread_count:
    • Keeps track of the number of active threads.
    • Ensures we don’t exceed the maximum number of allowed threads (MAX_THREADS).
  2. Modified quickSort Function:
    • Checks if conditions are suitable for parallel execution:
      • The number of active threads is less than MAX_THREADS.
      • The size of the partition is larger than PARALLEL_THRESHOLD.
    • If conditions are met, it increments thread_count and launches two new threads for the left and right partitions.
    • Each thread decrements thread_count when it finishes.
    • Uses lambda functions to capture the necessary variables for the threads.
    • Uses std::thread::join() to wait for the threads to finish before proceeding.
  3. Adjusted Thresholds:
    • Increased INSERTION_SORT_THRESHOLD to 100 for better performance on small arrays.
    • Set PARALLEL_THRESHOLD to 10,000 to avoid overhead from parallelizing small partitions.
  4. No Need for Mutexes:
    • Since each thread works on a separate portion of the array, there’s no need for mutexes to protect shared data.

Considerations:

  • Thread Overhead:
    • Creating too many threads can lead to overhead that negates the benefits of parallelization.
    • The implementation carefully limits the number of threads to the number of hardware threads available.
  • Random Number Generator:
    • Passing rng by reference into threads is safe because std::mt19937 is not thread-safe, but each thread uses its own copy due to capturing by value in the lambda functions.
    • Alternatively, we can create a separate rng instance within each thread if needed.
  • Performance:
    • Parallelization should improve performance on multi-core processors, especially for large datasets.
    • The actual performance gain depends on the system’s hardware and current load.
